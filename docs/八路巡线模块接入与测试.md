# 八路巡线模块接入与测试

> 适用范围：本文是 **I2C 八路智能模块 + OLED** 的独立诊断工程说明，使用 `line_sensor_test_LP_MSPM0G3507_nortos_ticlang.projectspec`。当前 2025 巡线车主工程使用 X1~X8 的 GPIO 直连传感器和 AT8236 电机，运行、调参与烧录请改看 [2025 巡线车固件与调参手册](2025巡线车固件与调参手册.md)。两种方案的接线和固件不能混用。

本文记录 LP-MSPM0G3507、Yahboom 八路智能巡线模块和 0.96 英寸 I2C OLED 的已验证接法。当前测试固件只读取传感器并显示结果，不配置或启动电机。

## 软件环境

- Code Composer Studio 21
- MSPM0 SDK 2.10.00.04
- TI Arm Clang
- SysConfig 1.26.2 或 CCS 随附的兼容版本

## 八路模块接线

模块使用标有 `GND SDA SCL 5V` 的 I2C 接口，不使用 RX/TX 接口或 X1～X8 独立输出。

| 八路模块 | LP-MSPM0G3507 | BoosterPack 引脚 |
|---|---|---|
| 5V | 5V | J3.21 |
| GND | GND | J3.22 |
| SCL | PA15 / I2C1_SCL | J3.30 |
| SDA | PA16 / I2C1_SDA | J3.29 |

J15 必须保持选择 PA16，使 PA16 连接到 J3.29。模块的 I2C 地址为 `0x12`，数字结果位于寄存器 `0x30`。X1 对应 bit7，X8 对应 bit0；白底为 1，黑线为 0。

## OLED 接线

使用四针、128×64、SSD1306 I2C OLED。USB 接口朝上时：

| OLED | LP-MSPM0G3507 | BoosterPack 引脚 |
|---|---|---|
| VCC | 3V3 | J1.1 |
| GND | GND | J2.20 |
| SCL | PA27 | J1.8 |
| SDA | PA26 | J1.5 |

必须拔掉 J17、J18 跳线帽，断开板载光传感器电路对 PA27、PA26 的占用。不要拔掉八路模块所需的 J15。OLED 使用 3.3V 供电，避免其上拉电阻将信号线拉到 5V。

## 模块标定

1. 上电后等待至少 20 秒。
2. 长按模块 KEY1，红灯常亮后进入标定。
3. 全部探头置于黑线并保持 3 秒，短按 KEY1。
4. 全部探头置于白底并保持 3 秒，再短按 KEY1。
5. 红灯熄灭表示成功；慢闪表示失败。

标定应在最终安装高度和实际赛道光照条件下完成。

## 导入和烧录测试工程

1. 在 CCS 中选择 **File > Import Project(s)**。
2. 导入 `firmware/mspm0-vehicle/ccs/line_sensor_test_LP_MSPM0G3507_nortos_ticlang.projectspec`。
3. 构建 `line_sensor_test_LP_MSPM0G3507_nortos_ticlang`。
4. 通过板载 XDS110 的 Micro-USB 接口连接开发板。
5. 点击 Debug；若下载后停在 `main()`，点击 Resume。

正常情况下，OLED 显示 `I2C OK` 和 X1～X8 的实时 0/1 状态。显示 `I2C ERROR` 时检查 J15、PA15/PA16 和模块四根线；OLED 全黑时检查 3V3/GND、PA26/PA27，以及 J17/J18 是否已拔除。

## 实板验证范围

已验证模块标定、八路探头响应、I2C 读取和 OLED 实时显示。正式巡线前仍需依次验证电机方向、编码器符号和低速闭环控制，不能直接套用厂商四驱底盘的电机参数。
