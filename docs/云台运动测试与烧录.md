# 云台运动测试与烧录（新手操作版）

本指南只测试云台机械运动。现在未安装激光也没有关系；测试程序会始终把 PB6 激光控制脚置为低电平。它仍会初始化串口接收，但不会依据相机、车辆或惯导数据驱动云台，也不会进入自动扫描。

## 从源码生成测试固件

仓库不提交 `build/` 目录或 `.hex` 文件，因此不能下载仓库之外本机生成的测试固件。请按本指南启用测试开关、重新编译，再将刚生成的 HEX 下载到控制板。这样可以确保脉冲数、方向和测试开关都与当前源码一致。

## 你需要准备的东西

1. 一台 Windows 电脑。
2. ST-LINK/V2 下载器，或开发板自带的 ST-LINK。
3. 四根杜邦线：`GND`、`SWDIO`、`SWCLK`，以及可选的 `NRST`。
4. 云台控制板的正常供电和电机驱动器的正常供电。
5. Keil MDK-ARM 5（用于重新生成测试程序），以及 STM32CubeProgrammer（用于下载，推荐安装）。

本工程目标芯片是 **STM32H750VBTx**。Keil 工程在 `firmware/stm32h750-gimbal/MDK-ARM/TMC2209.uvprojx`。

## 第 0 步：先保证机械安全

- 先不要安装激光；当前没有安装正适合做这一步。
- 让两个轴都离机械限位留有空间，至少能往两个方向各动一点。
- 第一次测试保持云台悬空或用手轻扶，但**不要阻挡、掰动或碰触正在运动的转轴**。
- 确认电机驱动器有单独的正确供电。ST-LINK 的 3.3 V 不能给电机供电。
- 遇到方向朝限位、卡住、异响或发烫，立即关闭**电机电源**，不要等程序自己结束。

## 第 1 步：启用本次运动测试

打开 [main.c](../firmware/stm32h750-gimbal/Core/Src/main.c)，在文件开头找到：

```c
#define GIMBAL_MOTION_TEST_ENABLED        0
```

只把最后的 `0` 改为 `1`，保存文件。先不要改其他参数。

测试程序会按下面的顺序运行一次，然后关闭两个电机驱动器并停住：

```text
使能驱动器
  → X 轴正向走 5 个脉冲 → 等待 1 秒 → X 轴反向返回
  → Y 轴正向走 5 个脉冲 → 等待 1 秒 → Y 轴反向返回
  → 失能驱动器，永久停住
```

5 个脉冲是刻意设得很小的安全初值。如果确认两轴都远离限位、方向正常但看不出位移，可将 `GIMBAL_TEST_X_STEPS` 或 `GIMBAL_TEST_Y_STEPS` 一次只增加到 `10`，重新编译和下载后再试。不要一开始改成几百或几千。

代码中的对应关系是：逻辑 X 轴用 `PA4`（STEP）和 `PA5`（DIR），逻辑 Y 轴用 `PA6`（STEP）和 `PA7`（DIR）。这不一定等于你肉眼看到的“左右/上下”方向；实际方向由电机安装决定。

## 第 2 步：连接 ST-LINK

在**控制板断电**时接线。ST-LINK 与控制板的 SWD 接线如下：

| ST-LINK 引脚 | 控制板引脚 | 说明 |
| --- | --- | --- |
| GND | GND | 必须共地 |
| SWDIO | PA13 / SWDIO | 数据线 |
| SWCLK | PA14 / SWCLK | 时钟线 |
| NRST（可选） | NRST / RESET | 下载连不上时很有用 |

控制板由自己的正常电源供电时，**不要接 ST-LINK 的 3.3V/5V 电源线**，只接上表中的信号和 GND。若你明确知道板子允许由 ST-LINK 3.3 V 供电，才接 3.3 V；无论如何都不要把 ST-LINK 的 5 V 接到 MCU 的 3.3 V 引脚。

接好后，先给控制板供电，再把 ST-LINK 插入电脑 USB。Windows 设备管理器能看到 ST-LINK 时，说明驱动基本正常。

## 第 3 步：用 Keil 编译测试程序

1. 安装 Keil MDK-ARM 5。首次打开工程若提示安装芯片包，安装 `Keil.STM32H7xx_DFP`；工程原本记录的版本是 `3.1.1`。
2. 双击打开 `firmware/stm32h750-gimbal/MDK-ARM/TMC2209.uvprojx`。
3. 按 `F7`，或点击工具栏的 **Build**。没有红色错误且底部显示 `0 Error(s)` 才算编译成功。
4. 成功后，新的下载文件在 `firmware/stm32h750-gimbal/MDK-ARM/TMC2209/TMC2209.hex`。

不要下载以前本机遗留的 `TMC2209.hex`；必须完成这一步重新生成，才能确保下载的固件包含本次运动测试改动。

如果 Keil 提示找不到 ARM 编译器、芯片包或有代码容量限制，先截图保存报错信息；不要随意删除工程设置。

## 第 4 步：下载到板子

### 方法 A：在 Keil 中一键下载

1. 菜单打开 **Flash → Configure Flash Tools**。
2. 在 **Debug** 页选择 `ST-Link Debugger`；点击 **Settings**，端口选择 `SW`（SWD）。
3. 在 **Utilities** 页勾选 `Use Target Driver for Flash Programming`，同样选择 `ST-Link Debugger`。
4. 点击工具栏的 **Load**，或按 `F8`。
5. 看到 `Flash Download: Programming Done` 表示下载完成。

若提示无法连接，先降低 ST-LINK 的 SWD 频率；仍不行时接上 `NRST`，并在按住板子复位键的同时点击下载，连接后再松开。

### 方法 B：用 STM32CubeProgrammer 下载 HEX

1. 安装并打开 STM32CubeProgrammer。
2. 左侧选择 **ST-LINK**，接口选择 **SWD**，点击 **Connect**。
3. 打开刚刚重新编译得到的 `firmware/stm32h750-gimbal/MDK-ARM/TMC2209/TMC2209.hex`。恢复正常程序时，将 `GIMBAL_MOTION_TEST_ENABLED` 改回 `0`，重新编译后再下载新生成的同名文件。
4. 点击 **Download**，等待成功提示。
5. 断开并重新给控制板上电，或按一次板子的复位键，测试才会重新执行。

## 第 5 步：观察并判断结果

上电后程序只执行一轮：先 X 轴往返，再 Y 轴往返，最后关闭驱动器。两个轴之间有约 1 秒停顿。

| 现象 | 优先检查 |
| --- | --- |
| 完全不动、无保持力 | 电机和驱动器供电；UART2/UART3 的驱动器使能线；连接器方向 |
| 只有一个轴动 | 对应轴的电机线、驱动器和 STEP/DIR 线 |
| 方向反了但能往返 | 记录是哪一轴反向；先不要改线，后续可在代码中统一改逻辑方向 |
| 往一个方向不回或撞限位 | 立即断开电机电源；将测试脉冲恢复到 5，检查机械空间和方向 |
| 每次运动角度不同 | 这是开环测试，检查驱动器细分、电流和机械阻力，不能直接开始 PID 跟踪 |

测试完成后，把 `GIMBAL_MOTION_TEST_ENABLED` 改回 `0`，重新编译并下载，才会恢复原来的菜单/扫描/跟踪程序。
